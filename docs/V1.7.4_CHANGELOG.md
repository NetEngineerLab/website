# NetEngineerLab V1.7.4 更新说明

## 版本目标

V1.7.4新增生产环境自动巡检系统，不增加工具，不修改现有12个工具的计算公式和输入输出逻辑。

## 新增功能

1. 新增`scripts/production-online-check.js`。
2. 新增`.github/workflows/production-online-monitor.yml`。
3. GitHub推送到`main`后自动等待Cloudflare Pages部署生效。
4. 每天北京时间08:15自动检查正式网站。
5. 使用本地文件SHA-256内容哈希判断线上是否已部署当前提交。
6. 检查主域名、`www`域名、36个Sitemap网址和8条旧网址跳转。
7. 检查12个在线工具、分类数量和24个中英文工具页面。
8. 检查工具目录中`tools-catalog.js`先于`site.js`加载。
9. 检查所有共享JavaScript和CSS的版本参数与实际文件内容一致。
10. 检查12个Service Worker与当前构建一致，并要求响应头重新验证。
11. 将完整结果写入`docs/PRODUCTION_ONLINE_REPORT.json`并上传为GitHub Actions构件。
12. 在GitHub Actions摘要中显示检查状态、部署等待次数、耗时及错误原因。

## 首次上线修正

1. 部署就绪检查由首页和工具目录扩大为36个Sitemap页面、8个共享资源及12个Service Worker。
2. 每次重试使用独立探测参数，避免Cloudflare分节点传播期间读取到新旧文件混合状态。
3. 完整巡检复用已经确认就绪的同一批响应，避免就绪后再次请求产生部署竞态。
4. Service Worker响应头路径改用精确的`/tools/:tool/sw.js`规则。

## 自动运行时机

- 每次推送到`main`分支。
- 每天北京时间08:15。
- GitHub Actions页面手动运行。

## 本地测试

```text
npm run prepare:launch
npm run preview
npm run check:online -- --base=http://127.0.0.1:4173
```

## 正式环境手动巡检

```text
npm run check:online -- --base=https://netengineerlab.com
```

## 验收标准

- `status`必须为`PASS`。
- Sitemap网址数量必须为36。
- 在线工具数量必须为12。
- 中英文工具页面数量必须为24。
- 共享静态资源和Service Worker必须与当前Git提交一致。
- 错误列表必须为空。
