Core calculation

Payload rate = physical rate × payload efficiency × allowed utilization. Transfer time = file bits ÷ payload rate + setup delay.

METHODOLOGY

Methodology

Bits and bytes

Network speed is expressed in bits per second, while file size is normally expressed in bytes. One byte equals eight bits.

Efficiency and utilization

Protocol efficiency represents the payload fraction. Utilization limits the share of physical link capacity allowed for the workload.

Concurrent transfers

The transfer-time mode assumes concurrent jobs share the available payload rate equally. Scheduling and congestion control can differ.

Latency and setup

Setup delay is modeled as RTT multiplied by setup round trips, then added to serialization time.

Unit references: SI Brochure · NIST SI Units · RFC 6349

FAQ

Frequently asked questions

Why does 100 Mbps not transfer 100 MB every second?

100 Mbps equals 12.5 MB/s before protocol and utilization losses.

Should file size use GB or GiB?

Use GB for decimal storage values and GiB when the source reports IEC binary units.

What efficiency should be used?

Use measured application throughput when available; otherwise start with a preset and retain margin.

Does RTT matter for large transfers?

The tool adds setup delay, while real TCP can also be limited by window growth and recovery.

How is maximum user count calculated?

Available payload bandwidth is divided by engineered per-user bitrate after the burst factor.

Can this replace a traffic-engineering model?

No. Queueing, QoS, retransmissions, traffic distribution and peak-hour behavior still need validation.
