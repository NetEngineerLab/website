METHODOLOGY

计算方法

CIDR计算

前缀长度生成连续子网掩码，网络地址通过按位与计算，广播地址通过网络地址与反掩码组合计算。

/31与/32

/31按两地址点到点链路处理，/32表示单一主机路由。

VLSM分配

子网需求按容量从大到小排序，并对齐到下一合法网络边界后依次分配。

IPv6前缀

IPv6使用128位整数运算，输出网络地址、最后地址、主机位以及相对上级前缀的子网数量。

主要标准来源： RFC 1918 · RFC 3021 · RFC 4632 · RFC 4291 · RFC 4193

FAQ

常见问题

/24有多少可用主机？

/24共有256个地址，通常可用主机地址为254个。

为什么/31显示2个可用地址？

RFC 3021允许点到点链路使用两个地址，不再保留传统网络地址和广播地址。

什么是反掩码？

反掩码是子网掩码的按位取反，常用于ACL和路由配置。

VLSM是否自动预留网关地址？

每个子网都有可编辑预留字段，可用于网关、冗余地址、基础设施和后续扩容。

IPv6地址数量能否精确计算？

可以。本工具使用BigInt整数运算，并显示完整十进制地址数量。

只要地址够用就应该使用大广播域吗？

不应只看地址容量，还要考虑故障域、广播流量、安全区域、路由汇总和运维边界。
